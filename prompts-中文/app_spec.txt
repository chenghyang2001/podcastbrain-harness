<project_specification>
  <project_name>PodcastBrain — Podcast 與 YouTube 知識萃取平台</project_name>

  <overview>
    PodcastBrain 是一個 Streamlit 網頁應用程式，能將 Podcast 集數和 YouTube 影片
    轉換成結構化的知識資產。使用者提供 URL（YouTube、直接 MP3/M4A 連結）或上傳
    音頻檔案，應用程式會自動以 yt-dlp 下載音頻、使用 OpenAI Whisper 在本機進行轉錄
    （音頻絕不離開本機），然後使用 Claude AI 生成章節、多段落摘要、帶時間戳記的關鍵
    引述、行動項目與說話者識別。互動式問答模式讓使用者提問，答案完全來自集數逐字稿，
    每個回應都有逐字稿原文為根據。

    應用程式以隱私為優先：所有轉錄均在本機使用本地 Whisper 模型完成。
    除了送給 Claude 分析的純文字外，音頻或原始逐字稿不會傳送至任何外部服務，
    Claude 也只接收文字。全文搜尋媒體庫（SQLite FTS5）為所有逐字稿建立索引，
    讓使用者可在毫秒內搜尋整個集數媒體庫。批次處理佇列按順序處理多個集數，
    穩健的匯出系統可產生 Markdown 筆記、PDF 文件、SRT 字幕檔和純文字逐字稿。

    PodcastBrain 專為知識工作者、研究人員和學習者設計，讓他們無需手動記筆記即可
    從音頻內容中萃取最大價值。集數媒體庫隨時間增長，每個過去的集數都可隨時搜尋
    和重新分析。
  </overview>

  <core_features>
    1. 音頻輸入 — URL 輸入（YouTube 觀看網址、直接 MP3/M4A/WAV 連結）和
       檔案上傳小工具（.mp3、.m4a、.wav、.ogg）。yt-dlp 將音頻下載至暫存目錄。
       進度列從 yt-dlp stdout 解析更新。取消按鈕終止子程序。

    2. 本機 Whisper 轉錄 — 模型選擇下拉選單（tiny/base/small/medium/large），
       包含每個模型的預估時間和精確度說明。透過子程序呼叫 Whisper（whisper CLI 或
       Python API）。從片段數量估算進度。輸出完整文字和帶時間戳記的 JSON 片段。
       語言自動偵測或由使用者指定。

    3. Claude 章節偵測 — Claude 接收完整逐字稿文字，回傳 JSON 陣列：
       [{title, start_seconds, end_seconds, summary}, ...]。短集數退回
       （< 5 分鐘）：單一章節涵蓋完整時長。章節儲存至資料庫並顯示為可點擊時間軸。

    4. Claude 完整分析 — 三個獨立 Claude API 呼叫：(a) 摘要（3-5 段落含
       關鍵主題），(b) 關鍵引述（5-7 個含說話者歸因和時間戳記）、行動項目
       （具體後續步驟的編號清單）。(c) 說話者識別（依逐字稿中的語音模式描述
       識別說話者）。

    5. 向集數提問（問答功能）— 關鍵字片段檢索找到相關逐字稿段落。
       Claude 嚴格根據檢索到的片段回答——「僅根據逐字稿引用」系統提示。
       引用模式顯示使用的確切逐字稿段落。每個集數的問答歷史儲存至資料庫。
       聊天式 UI 帶輸入框和可滾動歷史記錄。

    6. 匯出選項 — Markdown（.md，含完整結構：摘要、章節、引述、逐字稿）、
       PDF（reportlab，帶區段標題格式）、SRT 字幕檔（來自 Whisper 片段）、
       純 TXT（僅逐字稿）、ZIP 壓縮檔（包含所有格式）。全部透過
       st.download_button 提供。

    7. 含搜尋功能的集數媒體庫 — FTS5 全文搜尋透過 st.text_input 跨所有集數
       逐字稿執行。依日期範圍、時長範圍和來源網域篩選。集數卡片格線顯示縮圖
       （YouTube）、標題、時長、處理日期。點擊卡片開啟集數詳情。

    8. 批次處理佇列 — 將多個 URL 加入佇列。依序處理（一次一個）。
       狀態表：[url, title, status, progress, eta]。狀態值：queued/downloading/
       transcribing/analyzing/complete/failed。選用電子郵件通知（透過 smtplib）
       於完成時發送。佇列透過 SQLite 在頁面重新載入後持續保存。
  </core_features>

  <database_schema>
    -- 主要集數記錄
    CREATE TABLE episodes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT,
        source_url TEXT NOT NULL,
        source_type TEXT CHECK(source_type IN ('youtube','direct_url','file_upload')),
        audio_path TEXT,                  -- 已下載音頻檔案的暫存路徑
        duration_secs INTEGER,
        whisper_model TEXT,               -- 例如 'base'、'small'
        language TEXT DEFAULT 'en',
        status TEXT CHECK(status IN ('queued','downloading','transcribing','analyzing','complete','failed')),
        error_message TEXT,
        claude_analysis TEXT,             -- JSON：{summary, quotes, action_items, speakers}
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        processed_at TIMESTAMP
    );

    -- 含片段級時間戳記的完整逐字稿
    CREATE TABLE transcripts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        full_text TEXT NOT NULL,
        segments TEXT,                    -- JSON 陣列：[{start, end, text}, ...]
        word_count INTEGER,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- 來自 Claude 分析的章節標記
    CREATE TABLE chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        title TEXT NOT NULL,
        start_seconds INTEGER NOT NULL,
        end_seconds INTEGER,
        summary TEXT,
        sort_order INTEGER NOT NULL
    );

    -- 每個集數的問答互動歷史記錄
    CREATE TABLE qa_history (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        question TEXT NOT NULL,
        answer TEXT NOT NULL,
        source_ts TEXT,                   -- 使用的時間戳記對 JSON 陣列：[[start, end], ...]
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- 全文逐字稿搜尋的 FTS5 虛擬資料表
    -- 逐字稿儲存後透過觸發器或手動插入來填充
    CREATE VIRTUAL TABLE transcripts_fts USING fts5(
        episode_id UNINDEXED,
        full_text,
        content='transcripts',
        content_rowid='id'
    );

    -- 保持 FTS5 同步的觸發器
    CREATE TRIGGER transcripts_ai AFTER INSERT ON transcripts BEGIN
        INSERT INTO transcripts_fts(rowid, episode_id, full_text)
        VALUES (new.id, new.episode_id, new.full_text);
    END;
  </database_schema>

  <key_interactions>
    流程 1 — 處理新集數：
      使用者輸入 YouTube 網址 → 點擊「處理集數」→ 側邊欄顯示 Whisper 模型選擇
      → 應用程式顯示 st.status() 容器，帶即時更新：「下載音頻中...」、「轉錄中
      （可能需要幾分鐘）...」、「使用 Claude 分析中...」→ 集數檢視器出現，
      所有 4 個分頁均已填充 → 使用者閱讀摘要並探索章節。

    流程 2 — 向集數提問：
      使用者點擊問答分頁 → 在文字輸入框中輸入問題 → 點擊「提問」→ qa_engine.py
      以關鍵字比對搜尋逐字稿片段 → Claude 僅根據檢索到的段落回答 →
      回應出現，附「來源：」引用，顯示使用的確切逐字稿摘錄 →
      問答對加入下方可滾動的聊天歷史記錄。

    流程 3 — 搜尋集數媒體庫：
      使用者在側邊欄導航中點擊「集數媒體庫」→ 在文字輸入框中輸入搜尋關鍵字 →
      FTS5 查詢對所有逐字稿執行 → 比對的集數卡片出現，帶高亮片段顯示搜尋詞
      出現的位置 → 使用者點擊卡片，開啟該集數的完整集數檢視器
      （從資料庫載入，無需重新處理）。
  </key_interactions>

  <implementation_steps>
    步驟 1 — 專案架構：
    建立目錄結構：podcastbrain/app.py、downloader.py、transcriber.py、
    analyzer.py、qa_engine.py、db.py、requirements.txt、README.md。
    實作 db.py：SQLAlchemy 2.x 引擎、所有 4 個 ORM 模型（Episode、Transcript、
    Chapter、QAHistory）、透過 text() DDL 建立 FTS5 虛擬資料表、插入後觸發器、
    FTS5 DDL 單獨執行的 create_all()、帶上下文管理器的 session factory。
    實作 Streamlit app.py 骨架：側邊欄導航（處理新集數 / 媒體庫 / 批次）、
    Whisper 模型 selectbox、語言 selectbox、最近集數清單。

    步驟 2 — yt-dlp 下載模組：
    實作 downloader.py，含 download_audio(url, output_dir) 函式。
    使用子程序呼叫 yt-dlp，參數包含：--format bestaudio、--extract-audio、
    --audio-format mp3、--progress、--output 範本。從 stdout 行解析進度百分比
    （正規表達式：r'\[download\]\s+(\d+\.\d+)%'）。
    透過回呼或輪詢進度檔案更新 Streamlit 進度列。
    使用 subprocess.terminate() 實作 cancel_download(pid)。
    處理：網路錯誤、私人影片、年齡限制內容、無效網址。
    成功時回傳：{file_path, title, duration_secs, thumbnail_url}。

    步驟 3 — Whisper 轉錄模組：
    實作 transcriber.py，含 transcribe_audio(audio_path, model_name, language) 函式。
    透過 Python API 呼叫 whisper：import whisper; model = whisper.load_model(model_name);
    result = model.transcribe(audio_path, language=language)。
    擷取：result["text"]（完整逐字稿）、result["segments"]（含 start/end/text 的清單）。
    進度估算：以音頻時長對比已用時間估算。
    處理：找不到檔案、不支援的格式、模型下載失敗、CUDA/CPU 選擇。
    回傳：{full_text, segments, word_count, language_detected}。

    步驟 4 — Claude 分析模組：
    實作 analyzer.py，含三個獨立 Claude API 函式：
    (a) generate_chapters(transcript_text, duration_secs, api_key) — 系統提示指示
    Claude 回傳含 title/start_seconds/end_seconds/summary 的章節 JSON 陣列。
    驗證 JSON 結構。解析失敗或時長 < 300 秒時退回至單一章節。
    (b) generate_summary_quotes_actions(transcript_text, api_key) — 回傳 JSON，含：
    summary（字串）、quotes（{text, speaker, timestamp_seconds} 陣列）、
    action_items（字串陣列）。
    (c) identify_speakers(transcript_text, api_key) — 回傳含說話者標籤的 JSON。
    三個呼叫均使用重試機制（3 次嘗試，指數退避）。將合併結果以 JSON 形式
    儲存於 episodes.claude_analysis 欄位。

    步驟 5 — Streamlit UI：輸入與處理：
    「處理新集數」頁面：URL 文字輸入 + 檔案上傳小工具（接受 .mp3、.m4a、.wav、.ogg）。
    「處理集數」按鈕觸發：在資料庫建立狀態為 'queued' 的 Episode 記錄，
    然後循序執行：下載 → 轉錄 → 分析 → 每步更新狀態。
    使用 st.status() 容器（Streamlit 1.28+）顯示即時步驟更新，含可展開詳情。
    顯示 Whisper 模型資訊：tiny（最快，約 95% 精確度）、base（良好平衡）、medium（高精確度）。
    處理完成後自動渲染集數檢視器。

    步驟 6 — Streamlit UI：集數檢視器：
    每個集數的四分頁佈局：摘要、章節、逐字稿、問答。
    摘要分頁：來自 claude_analysis JSON 的格式化 Markdown。關鍵引述以引言區塊顯示，
    含時間戳記。行動項目為帶核取方塊的編號清單（st.checkbox）。
    章節分頁：水平時間軸視覺化（帶章節片段的 Plotly 長條圖）。
    下方章節卡片：標題、「MM:SS - MM:SS」、摘要文字。
    逐字稿分頁：可滾動容器中的完整逐字稿。每個片段前以暗色文字顯示時間戳記。
    使用 st.text_input + 高亮比對文字進行集數內搜尋。
    問答分頁：帶 st.chat_input（Streamlit 1.31+）或 st.text_input 的聊天介面。
    聊天歷史以 st.chat_message 容器渲染。來源引用在可摺疊的 st.expander 中展示。

    步驟 7 — 問答引擎：
    實作 qa_engine.py，含 answer_question(episode_id, question, api_key, db_session)。
    片段檢索：將問題拆分為關鍵字，在逐字稿片段 JSON 中搜尋關鍵字比對，
    按關鍵字比對分數回傳前 10 名最相關片段。
    建立 Claude 提示：系統 = 「僅根據提供的逐字稿摘錄回答。
    如果答案不在逐字稿中，請明確說明。不得憑空捏造。」
    使用者訊息 = 問題 + 以編號摘錄含時間戳記格式化的檢索片段。
    解析回應：擷取答案文字並識別引用的片段編號。
    將問答對連同來源時間戳記對的 JSON 陣列儲存至 qa_history 資料表。
    回傳：{answer, cited_segments, raw_response}。

    步驟 8 — 媒體庫、搜尋、批次佇列與匯出：
    媒體庫頁面：st.text_input 觸發 FTS5 查詢：SELECT e.*, snippet(transcripts_fts, ...)。
    三欄格線中的集數卡片：標題、source_url 網域、時長、processed_at。
    透過 st.date_input 進行日期範圍篩選。透過 st.slider 進行時長篩選（0-180 分鐘）。
    批次佇列頁面：st.text_area 用於多 URL 輸入（每行一個）。「加入佇列」按鈕
    以 status='queued' 插入 Episode 記錄。狀態表使用 st.empty() + time.sleep 迴圈
    或 st.rerun() 每 5 秒自動重新整理。依序處理佇列。
    app.py 中的匯出函式：export_markdown(episode_id)、export_pdf(episode_id)、
    export_srt(episode_id)、export_txt(episode_id)。ZIP 透過 Python zipfile 模組。
    所有匯出透過 st.download_button 提供，帶適當的 MIME 類型。
