  <project_name>PodcastBrain — v3: 完整分析 + 問答功能</project_name>

  <overview>
    PodcastBrain v3 在 v2「下載-轉錄-章節」基礎上，新增兩個最具價值的智慧層：
    Claude 完整分析與互動式問答（Q&A）。處理完成後，使用者可在四標籤頁的「劇集檢視器」中查看：
    摘要（多段落概覽、關鍵引言、行動項目、發言人識別）、章節、逐字稿，以及問答。
    問答標籤頁讓使用者提問，所有答案均嚴格根據該集的逐字稿內容作答——
    不會產生幻覺，每個答案都有實際說話內容為依據。所有資料（含每集的問答紀錄）均持久化儲存於 SQLite。
  </overview>

  <core_features>
    1. 音訊輸入 — URL 文字輸入（YouTube 觀看 URL、直接 MP3/M4A/WAV 連結）與
       檔案上傳元件（.mp3、.m4a、.wav）。yt-dlp 將音訊下載至暫存目錄。
       透過解析 stdout 更新進度條。取消按鈕可終止子程序。

    2. 本地 Whisper 語音轉錄 — 模型選擇下拉選單（tiny/base/small/medium/large）。
       透過 Python API 呼叫 Whisper。擷取 full_text 與 segments（start/end/text）。
       語言自動偵測或由使用者指定。結果儲存於 transcripts 資料表。

    3. Claude 章節偵測 — Claude 接收完整逐字稿文字，回傳 JSON 陣列：
       [{title, start_seconds, end_seconds, summary}, ...]。JSON 驗證後，
       以指數退避策略重試 3 次。若時長少於 300 秒則退回單一章節。
       章節資料儲存於資料庫。

    4. Claude 完整分析 — analyzer.py 中三次獨立的 Claude API 呼叫：
       (a) generate_summary_quotes_actions(transcript_text, api_key) — 回傳 JSON，包含：
           summary（3-5 段摘要）、quotes（{text, speaker, timestamp_seconds} 陣列）、
           action_items（具體後續步驟的編號清單）。
       (b) identify_speakers(transcript_text, api_key) — 回傳 JSON，包含發言人標籤
           與模式描述。
       所有呼叫均使用重試邏輯（3 次嘗試，指數退避）。合併結果以 JSON 格式
       儲存於 episodes.claude_analysis 欄位。

    5. 向本集提問（Q&A） — qa_engine.py：從逐字稿 JSON 中以關鍵字檢索片段。
       最相關的前 10 個片段提供給 Claude。系統提示為：「僅根據提供的逐字稿摘錄回答。
       如果答案不在逐字稿中，請明確說明。不得產生幻覺。」
       回答包含引用的片段時間戳。問答對儲存於 qa_history 資料表。
       聊天式 UI 含可捲動的歷史記錄。
  </core_features>

  <database_schema>
    -- 主要劇集記錄
    CREATE TABLE episodes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT,
        source_url TEXT NOT NULL,
        source_type TEXT CHECK(source_type IN ('youtube','direct_url','file_upload')),
        audio_path TEXT,
        duration_secs INTEGER,
        whisper_model TEXT,
        language TEXT DEFAULT 'en',
        status TEXT CHECK(status IN ('queued','downloading','transcribing','analyzing','complete','failed')),
        error_message TEXT,
        claude_analysis TEXT,   -- JSON: {summary, quotes, action_items, speakers}
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        processed_at TIMESTAMP
    );

    -- 含片段級時間戳的完整逐字稿
    CREATE TABLE transcripts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        full_text TEXT NOT NULL,
        segments TEXT,          -- JSON 陣列：[{start, end, text}, ...]
        word_count INTEGER,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- Claude 分析產生的章節標記
    CREATE TABLE chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        title TEXT NOT NULL,
        start_seconds INTEGER NOT NULL,
        end_seconds INTEGER,
        summary TEXT,
        sort_order INTEGER NOT NULL
    );

    -- 每集的問答互動歷史記錄
    CREATE TABLE qa_history (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        question TEXT NOT NULL,
        answer TEXT NOT NULL,
        source_ts TEXT,         -- 使用的時間戳對 [start, end] 的 JSON 陣列
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
  </database_schema>

  <key_interactions>
    流程 1 — 處理新劇集：
      使用者貼上 URL + 選擇 Whisper 模型 → 點擊「處理劇集」 →
      st.status() 顯示：「下載中...」、「轉錄中...」、「偵測章節中...」、
      「Claude 分析中...」→ 劇集檢視器的四個標籤頁全部填入資料。

    流程 2 — 提問：
      使用者開啟問答標籤頁 → 輸入問題 → 點擊「提問」→ qa_engine 以關鍵字搜尋逐字稿
      片段 → Claude 僅根據擷取的段落作答 → 回答顯示「來源：」引用
      （可展開，顯示使用的確切摘錄）→ 問答對追加至可捲動的聊天紀錄。

    流程 3 — 查看過去的劇集：
      使用者點擊「我的劇集」→ 列出所有劇集（含標題、日期、時長）→
      點擊可開啟劇集檢視器（從資料庫載入，所有標籤頁均可使用，無需重新處理）。
  </key_interactions>

  <implementation_steps>
    步驟 1 — 專案鷹架：
    建立：podcastbrain/app.py、downloader.py、transcriber.py、analyzer.py、qa_engine.py、
    db.py、requirements.txt。
    db.py：SQLAlchemy 引擎、Episode/Transcript/Chapter/QAHistory ORM 模型、create_all()。

    步驟 2 — yt-dlp 下載模組：
    downloader.py：download_audio(url, output_dir)。解析進度，支援取消。
    回傳 {file_path, title, duration_secs}。

    步驟 3 — Whisper 語音轉錄模組：
    transcriber.py：transcribe_audio(audio_path, model_name, language)。
    回傳 {full_text, segments, word_count, language_detected}。

    步驟 4 — Claude 分析模組：
    analyzer.py：三個獨立函式，含重試邏輯（3 次嘗試，指數退避）：
    (a) generate_chapters(transcript_text, duration_secs, api_key) — JSON [{title, start, end, summary}]
    (b) generate_summary_quotes_actions(transcript_text, api_key) — JSON {summary, quotes, action_items}
    (c) identify_speakers(transcript_text, api_key) — JSON {speakers: [...]}
    合併結果以 JSON 格式儲存於 episodes.claude_analysis。

    步驟 5 — 問答引擎：
    qa_engine.py：answer_question(episode_id, question, api_key, db_session)。
    拆分關鍵字 → 搜尋 segments JSON → 依匹配分數取前 10 名 → 以
    「僅根據逐字稿回答」的系統提示呼叫 Claude → 儲存至 qa_history → 回傳
    {answer, cited_segments}。

    步驟 6 — Streamlit UI：輸入與處理：
    「處理新劇集」頁面。循序執行：下載 → 轉錄 → 分析 → 更新狀態。
    st.status() 顯示即時步驟訊息。

    步驟 7 — Streamlit UI：劇集檢視器：
    四標籤頁佈局。摘要標籤：格式化 markdown、引言區塊、
    st.checkbox 行動項目。章節標籤：章節卡片，含標題、MM:SS-MM:SS、摘要。
    逐字稿標籤：可捲動完整文字，含淡色時間戳前綴。
    問答標籤：st.chat_input（或 st.text_input）+ 可捲動的聊天紀錄，
    使用 st.chat_message 容器。來源顯示於 st.expander。
  </implementation_steps>
