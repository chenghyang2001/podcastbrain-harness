  <overview>
    PodcastBrain v5 是完整的最終前版本規格，在 v4 的基礎上新增了最後一個主要功能——
    批次處理佇列。使用者現在可以一次貼上多個網址（每行一個）到批次佇列中，
    應用程式將依序處理，並以即時狀態表格顯示每個集數在下載 → 轉錄 → 分析
    管線中的進度。佇列狀態持久儲存於 SQLite，頁面重新載入後不會遺失佇列中的工作。
    批次處理完成時，可選擇性地透過 smtplib 發送電子郵件通知。
    章節分頁新增了以 Plotly 呈現的章節時間軸視覺化圖表。
    此版本在功能覆蓋範圍上與最終 app_spec.txt 完全相同，
    差異僅在於自主編碼代理人的實作細節和複雜度指引層次。
  </overview>

  <core_features>
    1. 音頻輸入 — URL 文字輸入（YouTube 觀看網址、直接 MP3/M4A/WAV 連結）和
       檔案上傳小工具（.mp3、.m4a、.wav、.ogg）。yt-dlp 將音頻下載至暫存目錄。
       進度列從 yt-dlp stdout 解析更新。取消按鈕終止子程序。

    2. 本機 Whisper 轉錄 — 模型選擇下拉選單（tiny/base/small/medium/large），
       含每個模型的預估時間和精確度說明。透過 Python API 呼叫 Whisper：
       import whisper; model = whisper.load_model(model_name);
       result = model.transcribe(audio_path, language=language)。
       擷取：result["text"]（完整逐字稿）、result["segments"]（含 start/end/text 的清單）。
       根據音頻時長與已用時間估算進度。
       處理：找不到檔案、不支援的格式、模型下載失敗、CUDA/CPU 選擇。

    3. Claude 章節偵測 — Claude 接收完整逐字稿文字，回傳 JSON 陣列：
       [{title, start_seconds, end_seconds, summary}, ...]。驗證 JSON 結構。
       短集數退回（少於 5 分鐘）：單一章節涵蓋完整時長。最多重試 3 次，指數退避。
       章節儲存至資料庫，以可點擊的時間軸呈現。

    4. Claude 完整分析 — 三個獨立 Claude API 呼叫：
       (a) generate_summary_quotes_actions(transcript_text, api_key) — 回傳 JSON：
           summary（3-5 段落含關鍵主題）、quotes（{text, speaker, timestamp_seconds} 陣列）、
           action_items（具體後續步驟的編號清單）。
       (b) identify_speakers(transcript_text, api_key) — 回傳含說話者標籤的 JSON。
       所有呼叫使用重試機制（最多 3 次，指數退避）。合併結果以 JSON 形式
       儲存於 episodes.claude_analysis 欄位。

    5. 向集數提問（問答功能） — 關鍵字片段檢索找到相關的逐字稿段落。
       Claude 嚴格根據檢索到的片段回答——「僅引用逐字稿」系統提示。
       引用模式顯示所使用的確切逐字稿段落。問答歷史儲存至每個集數的資料庫中。
       含 st.chat_input 和可滾動歷史記錄（使用 st.chat_message）的聊天式 UI。

    6. 匯出選項 — Markdown（.md）含完整結構（摘要、章節、引述、逐字稿），
       PDF（reportlab，含章節標題格式）、SRT 字幕檔（來自 Whisper 片段）、
       純文字（僅逐字稿）、包含所有格式的 ZIP 壓縮檔。全部透過 st.download_button 實現。

    7. 含搜尋功能的集數媒體庫 — 透過 st.text_input 對所有集數逐字稿進行 FTS5 全文搜尋。
       以 st.date_input 篩選日期範圍，st.slider 篩選時長範圍（0-180 分鐘）。
       3 欄集數卡片格網，含縮圖、標題、時長、處理日期。點擊卡片
       開啟該集數的集數檢視器（從資料庫載入，無需重新處理）。
       snippet() 函式在搜尋結果中提供高亮摘錄。

    8. 批次處理佇列 — 透過 st.text_area 新增多個網址（每行一個）。
       「加入佇列」按鈕以 status='queued' 插入集數記錄。
       循序處理（每次一個，避免資源競爭）。狀態表格顯示：網址、標題、狀態、
       進度、預估剩餘時間。狀態值：queued/downloading/transcribing/analyzing/complete/failed。
       使用 st.empty() + st.rerun() 每 5 秒自動更新表格。
       批次處理完成時可選擇性地透過 smtplib 發送電子郵件通知。
       佇列透過 SQLite 中的 status 欄位在頁面重新載入後持久保存。
  </core_features>

  <database_schema>
    -- 主要集數記錄
    CREATE TABLE episodes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT,
        source_url TEXT NOT NULL,
        source_type TEXT CHECK(source_type IN ('youtube','direct_url','file_upload')),
        audio_path TEXT,                  -- 下載的音頻檔案的暫存路徑
        duration_secs INTEGER,
        whisper_model TEXT,               -- 例如 'base'、'small'
        language TEXT DEFAULT 'en',
        status TEXT CHECK(status IN ('queued','downloading','transcribing','analyzing','complete','failed')),
        error_message TEXT,
        claude_analysis TEXT,             -- JSON：{summary, quotes, action_items, speakers}
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        processed_at TIMESTAMP
    );

    -- 含片段級時間戳記的完整逐字稿
    CREATE TABLE transcripts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        full_text TEXT NOT NULL,
        segments TEXT,                    -- JSON 陣列：[{start, end, text}, ...]
        word_count INTEGER,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- 來自 Claude 分析的章節標記
    CREATE TABLE chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        title TEXT NOT NULL,
        start_seconds INTEGER NOT NULL,
        end_seconds INTEGER,
        summary TEXT,
        sort_order INTEGER NOT NULL
    );

    -- 每個集數的問答互動歷史記錄
    CREATE TABLE qa_history (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        question TEXT NOT NULL,
        answer TEXT NOT NULL,
        source_ts TEXT,                   -- 使用的時間戳記對 JSON 陣列
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- FTS5 虛擬資料表（全文逐字稿搜尋）
    -- 在儲存逐字稿後透過觸發器或手動插入填充
    CREATE VIRTUAL TABLE transcripts_fts USING fts5(
        episode_id UNINDEXED,
        full_text,
        content='transcripts',
        content_rowid='id'
    );

    -- 觸發器，保持 FTS5 同步
    CREATE TRIGGER transcripts_ai AFTER INSERT ON transcripts BEGIN
        INSERT INTO transcripts_fts(rowid, episode_id, full_text)
        VALUES (new.id, new.episode_id, new.full_text);
    END;
  </database_schema>

  <key_interactions>
    流程 1 — 處理新集數：
      使用者輸入 YouTube 網址 → 點擊「處理集數」→ st.status() 顯示即時更新：
      「下載音頻中...」、「轉錄中（可能需要幾分鐘）...」、「使用 Claude 分析中...」
      → 集數檢視器出現，四個分頁均完整呈現 → 匯出按鈕可見。

    流程 2 — 向集數提問：
      使用者點擊問答分頁 → 輸入問題 → 點擊「提問」→ qa_engine.py 以關鍵字搜尋
      逐字稿片段 → Claude 僅根據檢索到的段落回答 → 回應顯示「來源：」引用
      在 st.expander 中 → 問答對追加至可滾動聊天歷史記錄。

    流程 3 — 搜尋集數媒體庫：
      使用者點擊「集數媒體庫」→ 輸入搜尋查詢 → FTS5 對所有逐字稿執行查詢
      → 含高亮文字摘錄的符合集數卡片出現 → 點擊卡片從資料庫開啟完整集數檢視器。

    流程 4 — 批次處理多個網址：
      使用者點擊「批次佇列」→ 貼上多個網址（每行一個）→ 點擊「加入佇列」→
      點擊「開始處理」→ 狀態表格每 5 秒更新，顯示每個集數的進度
      → 批次完成後發送電子郵件通知（若已設定）。
  </key_interactions>

  <implementation_steps>
    步驟 1 — 專案架構：
    建立：podcastbrain/app.py、downloader.py、transcriber.py、analyzer.py、qa_engine.py、
    db.py、requirements.txt、README.md。
    db.py：SQLAlchemy 2.x 引擎、4 個 ORM 模型、FTS5 虛擬資料表（透過 text() DDL）、
    插入後觸發器、create_all() 後單獨執行 FTS5 DDL、Session 工廠。
    app.py 骨架：側邊欄導航、Whisper 模型 selectbox、語言 selectbox、最近集數清單。

    步驟 2 — yt-dlp 下載模組：
    downloader.py：download_audio(url, output_dir)。子程序、進度解析、取消支援。
    處理：網路錯誤、私人影片、無效網址。
    回傳 {file_path, title, duration_secs, thumbnail_url}。

    步驟 3 — Whisper 轉錄模組：
    transcriber.py：transcribe_audio(audio_path, model_name, language)。
    Python API。回傳 {full_text, segments, word_count, language_detected}。

    步驟 4 — Claude 分析模組：
    analyzer.py：generate_chapters()、generate_summary_quotes_actions()、identify_speakers()。
    每個各含 3 次重試，指數退避。合併 JSON 儲存於 claude_analysis。

    步驟 5 — 問答引擎：
    qa_engine.py：answer_question(episode_id, question, api_key, db_session)。
    關鍵字拆分 → 片段搜尋 → 前 10 名 → Claude 呼叫含引用提示 → 儲存至 qa_history。

    步驟 6 — Streamlit UI：輸入與處理：
    「處理新集數」頁面。st.status() 顯示即時步驟訊息。完成後出現集數檢視器。

    步驟 7 — Streamlit UI：含匯出的集數檢視器：
    摘要分頁：格式化 Markdown、引言區塊引述、含 st.checkbox 的行動項目。
    章節分頁：Plotly 水平長條圖時間軸 + 章節卡片。
    逐字稿分頁：帶灰色時間戳記前綴的可滾動文字。
    問答分頁：st.chat_input + st.chat_message + 來源的 st.expander。
    匯出列：5 個 st.download_button 小工具（MD、PDF、SRT、TXT、ZIP）。

    步驟 8 — 媒體庫、搜尋、批次佇列：
