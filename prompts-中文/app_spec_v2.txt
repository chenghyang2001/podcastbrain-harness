  <project_name>PodcastBrain — v2：下載 + 轉錄 + 章節偵測</project_name>

  <overview>
    PodcastBrain v2 在最小可行版本的下載基礎上，新增了本機 Whisper 轉錄功能與
    Claude 驅動的章節偵測。使用者可以貼上 YouTube 網址、下載音頻、
    完全在裝置上完成轉錄（不向外部傳送音頻），然後讓 Claude 將逐字稿解析成
    帶有時間戳記的章節標題。結果顯示在兩分頁的集數檢視器中：
    逐字稿與章節。簡易 SQLite 資料庫在 Session 之間持久儲存集數，
    讓使用者無需重新處理已提交過的網址。
  </overview>

  <core_features>
    1. 音頻輸入 — URL 文字輸入（YouTube 觀看網址、直接 MP3/M4A/WAV 連結）和
       檔案上傳小工具（.mp3、.m4a、.wav）。yt-dlp 將音頻下載至暫存目錄。
       進度列從 yt-dlp stdout 解析更新。取消按鈕終止子程序。
       回傳：{file_path, title, duration_secs}。

    2. 本機 Whisper 轉錄 — 模型選擇下拉選單（tiny/base/small/medium/large），
       包含每個模型的預估時間說明。透過 Python API 呼叫 Whisper：
       import whisper; model = whisper.load_model(model_name);
       result = model.transcribe(audio_path)。
       擷取：result["text"]（完整逐字稿）和 result["segments"]（start/end/text）。
       兩者均儲存至資料庫。語言自動偵測。進度根據已用時間與音頻時長估算。

    3. Claude 章節偵測 — Claude 接收完整逐字稿文字，回傳 JSON 陣列：
       [{title, start_seconds, end_seconds, summary}, ...]。
       驗證 JSON 結構。解析失敗或音頻時長少於 300 秒時，退回至單一章節。
       章節儲存至資料庫，以包含 MM:SS 時間戳記的簡易章節清單呈現。
  </core_features>

  <database_schema>
    -- 主要集數記錄
    CREATE TABLE episodes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT,
        source_url TEXT NOT NULL,
        source_type TEXT CHECK(source_type IN ('youtube','direct_url','file_upload')),
        audio_path TEXT,
        duration_secs INTEGER,
        whisper_model TEXT,
        language TEXT DEFAULT 'en',
        status TEXT CHECK(status IN ('queued','downloading','transcribing','analyzing','complete','failed')),
        error_message TEXT,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        processed_at TIMESTAMP
    );

    -- 含片段級時間戳記的完整逐字稿
    CREATE TABLE transcripts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        full_text TEXT NOT NULL,
        segments TEXT,        -- JSON 陣列：[{start, end, text}, ...]
        word_count INTEGER,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- 來自 Claude 分析的章節標記
    CREATE TABLE chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        title TEXT NOT NULL,
        start_seconds INTEGER NOT NULL,
        end_seconds INTEGER,
        summary TEXT,
        sort_order INTEGER NOT NULL
    );
  </database_schema>

  <key_interactions>
    流程 1 — 處理新集數：
      使用者貼上網址 + 選擇 Whisper 模型 → 點擊「處理集數」→
      st.status() 顯示即時步驟：「下載中...」、「轉錄中...」、「偵測章節...」
      → 集數檢視器出現，包含逐字稿與章節兩個分頁。

    流程 2 — 查看過往集數：
      使用者點擊側邊欄「我的集數」→ 顯示先前處理過的集數清單 →
      點擊任一集數從資料庫開啟其集數檢視器（無需重新處理）。
  </key_interactions>

  <implementation_steps>
    步驟 1 — 專案架構：
    建立：podcastbrain/app.py、downloader.py、transcriber.py、analyzer.py、db.py、
    requirements.txt。
    db.py：SQLAlchemy 引擎、Episode/Transcript/Chapter ORM 模型、create_all()。

    步驟 2 — yt-dlp 下載模組：
    downloader.py：download_audio(url, output_dir)。子程序呼叫，進度解析，取消支援。
    回傳 {file_path, title, duration_secs}。

    步驟 3 — Whisper 轉錄模組：
    transcriber.py：transcribe_audio(audio_path, model_name, language)。
    Python API 呼叫。回傳 {full_text, segments, word_count, language_detected}。
    處理模型載入失敗和不支援的格式。

    步驟 4 — Claude 章節偵測模組：
    analyzer.py：generate_chapters(transcript_text, duration_secs, api_key)。
    系統提示指示 Claude 僅回傳 JSON 陣列。
    驗證 JSON 結構。失敗時最多重試 3 次。退回至單一章節。

    步驟 5 — Streamlit UI：輸入與處理：
    「處理新集數」頁面：URL text_input + 檔案上傳器。「處理集數」按鈕。
    循序執行：下載 → 轉錄 → 章節 → 更新資料庫狀態。
    使用 st.status() 容器顯示即時步驟訊息。

    步驟 6 — Streamlit UI：集數檢視器：
    兩分頁佈局：逐字稿（可滾動的完整文字含時間戳記）和章節
    （顯示標題、MM:SS 範圍、摘要的章節卡片）。
    「我的集數」頁面：簡易集數標題清單含處理日期，可點擊。
  </implementation_steps>

  <success_criteria>
    功能性：
    - 應用程式在 http://localhost:8501 載入
    - YouTube 網址完整處理：下載 → 轉錄 → 章節可見
    - 逐字稿分頁顯示含時間戳記的完整文字
    - 章節分頁顯示至少 1 個含標題和時間戳記的章節
    - 先前處理過的集數出現在「我的集數」中且可重新開啟
    - 無效網址顯示可讀的錯誤訊息（而非 Python 錯誤追蹤）

    技術品質：
    - 啟動時建立含正確資料表的 SQLite 資料庫
    - 以正確的模型名稱呼叫 Whisper 並回傳含時間戳記的片段
    - Claude 僅接收文字（絕不傳送音頻二進位資料）
    - analyzer.py 中的 generate_chapters() 包含重試與 JSON 退回機制
    - Claude API 金鑰從環境變數讀取，絕不硬編碼
  </success_criteria>
