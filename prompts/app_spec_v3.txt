  <project_name>PodcastBrain — v3: Full Analysis + Q&A</project_name>

  <overview>
    PodcastBrain v3 adds the two most valuable intelligence layers on top of v2's
    download-transcribe-chapters foundation: Claude Full Analysis and interactive
    Q&A. After processing, users see a four-tab Episode Viewer with Summary
    (multi-paragraph overview, key quotes, action items, speaker identification),
    Chapters, Transcript, and Q&A. The Q&A tab lets users ask questions that are
    answered exclusively from the episode's own transcript — no hallucination,
    every answer is grounded in what was actually said. All data persists in SQLite
    including Q&A history per episode.
  </overview>

  <core_features>
    1. Audio Input — URL text input (YouTube watch URLs, direct MP3/M4A/WAV links) and
       file upload widget (.mp3, .m4a, .wav). yt-dlp downloads audio to temp directory.
       Progress bar updated from stdout parsing. Cancel button terminates subprocess.

    2. Local Whisper Transcription — Model selection dropdown (tiny/base/small/medium/large).
       Whisper called via Python API. Extracts full_text and segments (start/end/text).
       Language auto-detected or user-specified. Results stored in transcripts table.

    3. Claude Chapter Detection — Claude receives full transcript text and returns JSON array:
       [{title, start_seconds, end_seconds, summary}, ...]. JSON validated, 3 retries with
       exponential backoff. Fallback to single chapter if duration less than 300 seconds.
       Chapters stored in DB.

    4. Claude Full Analysis — Three separate Claude API calls in analyzer.py:
       (a) generate_summary_quotes_actions(transcript_text, api_key) — returns JSON with:
           summary (3-5 paragraphs), quotes (array of {text, speaker, timestamp_seconds}),
           action_items (numbered list of concrete next steps).
       (b) identify_speakers(transcript_text, api_key) — returns JSON with speaker labels
           and pattern descriptions.
       All calls use retry logic (3 attempts, exponential backoff). Combined result stored
       as JSON in episodes.claude_analysis column.

    5. Ask the Episode Q&A — qa_engine.py: keyword-based segment retrieval from transcripts
       JSON. Top-10 most relevant segments fed to Claude. System prompt: "Answer only from
       the provided transcript excerpts. If the answer is not in the transcript, say so
       explicitly. Do not hallucinate." Response includes cited segment timestamps.
       Q&A pairs stored in qa_history table. Chat-like UI with scrollable history.
  </core_features>

  <database_schema>
    -- Primary episode record
    CREATE TABLE episodes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT,
        source_url TEXT NOT NULL,
        source_type TEXT CHECK(source_type IN ('youtube','direct_url','file_upload')),
        audio_path TEXT,
        duration_secs INTEGER,
        whisper_model TEXT,
        language TEXT DEFAULT 'en',
        status TEXT CHECK(status IN ('queued','downloading','transcribing','analyzing','complete','failed')),
        error_message TEXT,
        claude_analysis TEXT,   -- JSON: {summary, quotes, action_items, speakers}
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        processed_at TIMESTAMP
    );

    -- Full transcript with segment-level timestamps
    CREATE TABLE transcripts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        full_text TEXT NOT NULL,
        segments TEXT,          -- JSON array: [{start, end, text}, ...]
        word_count INTEGER,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- Chapter markers from Claude analysis
    CREATE TABLE chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        title TEXT NOT NULL,
        start_seconds INTEGER NOT NULL,
        end_seconds INTEGER,
        summary TEXT,
        sort_order INTEGER NOT NULL
    );

    -- Q&A interaction history per episode
    CREATE TABLE qa_history (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        question TEXT NOT NULL,
        answer TEXT NOT NULL,
        source_ts TEXT,         -- JSON array of [start, end] timestamp pairs used
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
  </database_schema>

  <key_interactions>
    Flow 1 — Process a new episode:
      User pastes URL + selects Whisper model → clicks "Process Episode" →
      st.status() shows: "Downloading...", "Transcribing...", "Detecting chapters...",
      "Analyzing with Claude..." → all four tabs populated in Episode Viewer.

    Flow 2 — Ask a question:
      User opens Q&A tab → types question → clicks "Ask" → qa_engine searches transcript
      segments by keyword → Claude answers from retrieved passages only → response shows
      with "Source:" citation (expandable, showing exact excerpt used) → Q&A pair appended
      to scrollable chat history.

    Flow 3 — View past episodes:
      User clicks "My Episodes" → list of episodes with title, date, duration →
      click to open Episode Viewer from DB (all tabs available, no re-processing).
  </key_interactions>

  <implementation_steps>
    Step 1 — Project Scaffold:
    Create: podcastbrain/app.py, downloader.py, transcriber.py, analyzer.py, qa_engine.py,
    db.py, requirements.txt.
    db.py: SQLAlchemy engine, Episode/Transcript/Chapter/QAHistory ORM models, create_all().

    Step 2 — yt-dlp Download Module:
    downloader.py: download_audio(url, output_dir). Progress parsing, cancel support.
    Returns {file_path, title, duration_secs}.

    Step 3 — Whisper Transcription Module:
    transcriber.py: transcribe_audio(audio_path, model_name, language).
    Returns {full_text, segments, word_count, language_detected}.

    Step 4 — Claude Analysis Module:
    analyzer.py: three separate functions with retry (3 attempts, exponential backoff):
    (a) generate_chapters(transcript_text, duration_secs, api_key) — JSON [{title, start, end, summary}]
    (b) generate_summary_quotes_actions(transcript_text, api_key) — JSON {summary, quotes, action_items}
    (c) identify_speakers(transcript_text, api_key) — JSON {speakers: [...]}
    Store combined result as JSON in episodes.claude_analysis.

    Step 5 — Q&A Engine:
    qa_engine.py: answer_question(episode_id, question, api_key, db_session).
    Keyword split → search segments JSON → top-10 by match score → Claude call with
    "Answer only from transcript" system prompt → store in qa_history → return
    {answer, cited_segments}.

    Step 6 — Streamlit UI: Input and Processing:
    Process New page. Sequential execution: download → transcribe → analyze → update status.
    st.status() for live step messages.

    Step 7 — Streamlit UI: Episode Viewer:
    Four-tab layout. Summary tab: formatted markdown, blockquote key quotes, action items
    with st.checkbox. Chapters tab: chapter cards with title, MM:SS-MM:SS, summary.
    Transcript tab: scrollable full text with dim timestamp prefixes.
    Q&A tab: st.chat_input (or st.text_input) + scrollable chat history with
    st.chat_message containers. Source in st.expander.
  </implementation_steps>
