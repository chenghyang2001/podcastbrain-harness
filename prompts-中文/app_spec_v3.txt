  <project_name>PodcastBrain — v3：完整分析 + 問答功能</project_name>

  <overview>
    PodcastBrain v3 在 v2 的下載-轉錄-章節基礎上，加入了兩個最具價值的智慧層：
    Claude 完整分析與互動式問答。處理完成後，使用者可在四分頁的集數檢視器中看到：
    摘要（多段落概述、關鍵引述、行動項目、說話者識別）、章節、逐字稿和問答。
    問答分頁讓使用者提問，答案完全來自該集數的逐字稿——零幻覺，
    每個回答都有逐字稿原文為根據。所有資料持久儲存於 SQLite，
    包括每個集數的問答歷史記錄。
  </overview>

  <core_features>
    1. 音頻輸入 — URL 文字輸入（YouTube 觀看網址、直接 MP3/M4A/WAV 連結）和
       檔案上傳小工具（.mp3、.m4a、.wav）。yt-dlp 將音頻下載至暫存目錄。
       進度列從 stdout 解析更新。取消按鈕終止子程序。

    2. 本機 Whisper 轉錄 — 模型選擇下拉選單（tiny/base/small/medium/large），
       包含每個模型的預估時間和精確度說明。透過 Python API 呼叫 Whisper。
       擷取 full_text 和 segments（start/end/text）。
       語言自動偵測或由使用者指定。結果儲存至 transcripts 資料表。

    3. Claude 章節偵測 — Claude 接收完整逐字稿文字，回傳 JSON 陣列：
       [{title, start_seconds, end_seconds, summary}, ...]。JSON 驗證，最多重試 3 次，
       指數退避。時長少於 300 秒時退回至單一章節。章節儲存至資料庫。

    4. Claude 完整分析 — analyzer.py 中三個獨立 Claude API 呼叫：
       (a) generate_summary_quotes_actions(transcript_text, api_key) — 回傳 JSON：
           summary（3-5 段落）、quotes（{text, speaker, timestamp_seconds} 陣列）、
           action_items（具體後續步驟的編號清單）。
       (b) identify_speakers(transcript_text, api_key) — 回傳含說話者標籤的 JSON。
       所有呼叫使用重試機制（最多 3 次，指數退避）。合併結果以 JSON 形式
       儲存於 episodes.claude_analysis 欄位。

    5. 向集數提問（問答功能） — qa_engine.py：關鍵字片段檢索（依比對分數排序前 10 名）。
       Claude 收到系統提示：「僅根據提供的逐字稿摘錄回答。如果答案不在逐字稿中，
       請明確說明。不得憑空捏造。」回應包含引用的片段時間戳記。
       問答對儲存至 qa_history 資料表。聊天式 UI 帶可滾動的歷史記錄。
  </core_features>

  <database_schema>
    -- 主要集數記錄
    CREATE TABLE episodes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT,
        source_url TEXT NOT NULL,
        source_type TEXT CHECK(source_type IN ('youtube','direct_url','file_upload')),
        audio_path TEXT,
        duration_secs INTEGER,
        whisper_model TEXT,
        language TEXT DEFAULT 'en',
        status TEXT CHECK(status IN ('queued','downloading','transcribing','analyzing','complete','failed')),
        error_message TEXT,
        claude_analysis TEXT,   -- JSON：{summary, quotes, action_items, speakers}
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        processed_at TIMESTAMP
    );

    -- 含片段級時間戳記的完整逐字稿
    CREATE TABLE transcripts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        full_text TEXT NOT NULL,
        segments TEXT,          -- JSON 陣列：[{start, end, text}, ...]
        word_count INTEGER,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- 來自 Claude 分析的章節標記
    CREATE TABLE chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        title TEXT NOT NULL,
        start_seconds INTEGER NOT NULL,
        end_seconds INTEGER,
        summary TEXT,
        sort_order INTEGER NOT NULL
    );

    -- 每個集數的問答互動歷史記錄
    CREATE TABLE qa_history (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        question TEXT NOT NULL,
        answer TEXT NOT NULL,
        source_ts TEXT,         -- 使用的時間戳記對 JSON 陣列：[[start, end], ...]
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
  </database_schema>

  <key_interactions>
    流程 1 — 處理新集數：
      使用者貼上網址 + 選擇 Whisper 模型 → 點擊「處理集數」→
      st.status() 顯示：「下載中...」、「轉錄中...」、「偵測章節...」、
      「使用 Claude 分析中...」→ 集數檢視器的四個分頁均完整呈現。

    流程 2 — 提問：
      使用者開啟問答分頁 → 輸入問題 → 點擊「提問」→ qa_engine 以關鍵字搜尋
      逐字稿片段 → Claude 僅根據檢索到的段落回答 → 回應顯示「來源：」引用
      （可展開，顯示使用的確切摘錄）→ 問答對追加至可滾動的聊天歷史記錄。

    流程 3 — 查看過往集數：
      使用者點擊「我的集數」→ 顯示集數清單含標題、日期、時長 →
      點擊後從資料庫開啟集數檢視器（所有分頁均可使用，無需重新處理）。
  </key_interactions>

  <implementation_steps>
    步驟 1 — 專案架構：
    建立：podcastbrain/app.py、downloader.py、transcriber.py、analyzer.py、qa_engine.py、
    db.py、requirements.txt。
    db.py：SQLAlchemy 引擎、Episode/Transcript/Chapter/QAHistory ORM 模型、create_all()。

    步驟 2 — yt-dlp 下載模組：
    downloader.py：download_audio(url, output_dir)。進度解析，取消支援。
    回傳 {file_path, title, duration_secs}。

    步驟 3 — Whisper 轉錄模組：
    transcriber.py：transcribe_audio(audio_path, model_name, language)。
    回傳 {full_text, segments, word_count, language_detected}。

    步驟 4 — Claude 分析模組：
    analyzer.py：三個獨立函式，各含重試機制（最多 3 次，指數退避）：
    (a) generate_chapters(transcript_text, duration_secs, api_key) — JSON [{title, start, end, summary}]
    (b) generate_summary_quotes_actions(transcript_text, api_key) — JSON {summary, quotes, action_items}
    (c) identify_speakers(transcript_text, api_key) — JSON {speakers: [...]}
    合併結果以 JSON 形式儲存於 episodes.claude_analysis。

    步驟 5 — 問答引擎：
    qa_engine.py：answer_question(episode_id, question, api_key, db_session)。
    關鍵字拆分 → 搜尋 segments JSON → 前 10 名 → Claude 呼叫含「僅根據逐字稿回答」
    系統提示 → 儲存至 qa_history → 回傳 {answer, cited_segments}。

    步驟 6 — Streamlit UI：輸入與處理：
    「處理新集數」頁面。循序執行：下載 → 轉錄 → 分析 → 更新狀態。
    st.status() 顯示即時步驟訊息。

    步驟 7 — Streamlit UI：集數檢視器：
    四分頁佈局。摘要分頁：格式化 Markdown、引言區塊引述、行動項目含 st.checkbox。
    章節分頁：含標題、MM:SS-MM:SS、摘要的章節卡片。
    逐字稿分頁：帶灰色時間戳記前綴的可滾動完整文字。
    問答分頁：st.chat_input（或 st.text_input）+ 含 st.chat_message 容器的可滾動聊天歷史。
    來源引用在 st.expander 中展示。
  </implementation_steps>
