  <project_name>PodcastBrain — v4：匯出功能 + 節目庫搜尋</project_name>

  <overview>
    PodcastBrain v4 在 v3 的基礎上新增兩項關鍵生產力功能：
    匯出選項，以及具備全文搜尋能力的節目庫。使用者現在可以
    將任何集數匯出為 Markdown、PDF、SRT 字幕或純文字格式，
    並一次下載包含所有格式的 ZIP 壓縮檔。節目庫頁面使用 SQLite
    FTS5，在毫秒內搜尋所有集數的逐字稿，並支援日期範圍與時長篩選。
    點擊搜尋結果可直接開啟該集數的完整節目檢視器，無需重新處理。
    這使得 PodcastBrain 成為音訊內容真正的個人知識庫，而不只是一次性的處理工具。
  </overview>

  <core_features>
    1. 音訊輸入 — URL 文字輸入（YouTube 觀看網址、直接 MP3/M4A/WAV 連結）及
       檔案上傳元件（.mp3、.m4a、.wav）。yt-dlp 下載音訊至暫存目錄。
       進度條由 stdout 解析更新。取消按鈕可終止子程序。

    2. 本地 Whisper 轉錄 — 模型選擇下拉選單（tiny/base/small/medium/large），
       附預估時間與準確度說明。透過 Python API 呼叫 Whisper。擷取
       full_text 及 segments（start/end/text）。語言自動偵測或由使用者指定。

    3. Claude 章節偵測 — Claude 接收完整逐字稿文字並回傳 JSON 陣列：
       [{title, start_seconds, end_seconds, summary}, ...]。JSON 已驗證，3 次重試搭配
       指數退避。對於短於 300 秒的集數，退回使用單一章節。

    4. Claude 完整分析 — 三次獨立的 Claude API 呼叫：
       (a) generate_summary_quotes_actions(transcript_text, api_key) — JSON {summary, quotes,
           action_items}，其中 quotes 包含 {text, speaker, timestamp_seconds}。
       (b) identify_speakers(transcript_text, api_key) — JSON {speakers: [...]}。
       合併結果以 JSON 形式儲存於 episodes.claude_analysis。

    5. 節目問答 — 以關鍵字為基礎的片段檢索（依匹配分數取前 10 名）。
       Claude 以「僅從逐字稿中回答」系統提示作答。來源引用
       顯示於 st.expander。問答配對儲存於 qa_history 資料表。

    6. 匯出選項 — export_markdown(episode_id)、export_pdf(episode_id)、
       export_srt(episode_id)、export_txt(episode_id)。ZIP 使用 Python zipfile 模組。
       Markdown 包含：摘要、章節、引述、逐字稿。PDF 透過 reportlab 加上
       章節標題。SRT 由 Whisper 片段產生，時間戳格式正確為 00:00:00,000。
       所有匯出均透過 st.download_button 搭配適當 MIME 類型提供。

    7. 具搜尋功能的節目庫 — 透過 st.text_input 以 FTS5 對所有集數逐字稿進行全文搜尋。
       以日期範圍（st.date_input）和時長（st.slider，0-180 分鐘）篩選。
       節目卡片網格（3 欄）：縮圖（YouTube）、標題、時長、
       處理日期。點擊卡片開啟該集數的節目檢視器（從資料庫載入，
       無需重新處理）。片段高亮顯示搜尋詞出現的位置。
  </core_features>

  <database_schema>
    -- 主要集數記錄
    CREATE TABLE episodes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT,
        source_url TEXT NOT NULL,
        source_type TEXT CHECK(source_type IN ('youtube','direct_url','file_upload')),
        audio_path TEXT,
        duration_secs INTEGER,
        whisper_model TEXT,
        language TEXT DEFAULT 'en',
        status TEXT CHECK(status IN ('queued','downloading','transcribing','analyzing','complete','failed')),
        error_message TEXT,
        claude_analysis TEXT,             -- JSON: {summary, quotes, action_items, speakers}
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        processed_at TIMESTAMP
    );

    -- 含片段級時間戳的完整逐字稿
    CREATE TABLE transcripts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        full_text TEXT NOT NULL,
        segments TEXT,                    -- JSON 陣列：[{start, end, text}, ...]
        word_count INTEGER,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- 來自 Claude 分析的章節標記
    CREATE TABLE chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        title TEXT NOT NULL,
        start_seconds INTEGER NOT NULL,
        end_seconds INTEGER,
        summary TEXT,
        sort_order INTEGER NOT NULL
    );

    -- 每集問答互動歷史
    CREATE TABLE qa_history (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        question TEXT NOT NULL,
        answer TEXT NOT NULL,
        source_ts TEXT,                   -- JSON 陣列，[start, end] 時間戳配對
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- FTS5 虛擬資料表，用於全文逐字稿搜尋
    CREATE VIRTUAL TABLE transcripts_fts USING fts5(
        episode_id UNINDEXED,
        full_text,
        content='transcripts',
        content_rowid='id'
    );

    -- 觸發器，保持 FTS5 同步
    CREATE TRIGGER transcripts_ai AFTER INSERT ON transcripts BEGIN
        INSERT INTO transcripts_fts(rowid, episode_id, full_text)
        VALUES (new.id, new.episode_id, new.full_text);
    END;
  </database_schema>

  <key_interactions>
    流程 1 — 處理新集數：
      URL → 處理 → st.status() 顯示四個步驟 → 節目檢視器含 4 個分頁 + 匯出列。

    流程 2 — 提問：
      問答分頁 → 輸入問題 → 含來源引用的有依據回答，顯示於 st.expander。

    流程 3 — 搜尋節目庫：
      節目庫頁面 → 輸入關鍵字 → FTS5 對所有逐字稿執行搜尋 → 節目卡片
      顯示並帶有片段高亮 → 點擊卡片從資料庫開啟完整節目檢視器。

    流程 4 — 匯出：
      任何集數 → 下載按鈕列 → Markdown / PDF / SRT / TXT / ZIP
      直接透過瀏覽器下載。
  </key_interactions>

    步驟 6 — 匯出模組：
    在 app.py 或 export.py 中：export_markdown(episode_id, db)、export_pdf(episode_id, db)、
    export_srt(episode_id, db)、export_txt(episode_id, db)。

    引述區塊、完整逐字稿。
    ZIP：在記憶體中建立 zipfile.ZipFile，寫入所有四種格式，回傳 BytesIO。
    全部透過 st.download_button 在節目檢視器中提供。

    步驟 7 — Streamlit UI：含匯出功能的節目檢視器：
    四分頁排版（摘要、章節、逐字稿、問答）。
    分頁下方放置 5 個 st.download_button 元件的匯出列。

    步驟 8 — Streamlit UI：節目庫：
    FTS5 查詢：SELECT e.*, snippet(transcripts_fts, 1, '**', '**', '...', 30)。
    3 欄網格顯示節目卡片。date_input 篩選日期範圍。滑桿篩選
    時長。點擊卡片設定 session_state episode_id 並切換至「處理新集數」
    頁面的節目檢視器模式。
