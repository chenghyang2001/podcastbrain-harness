  <project_name>PodcastBrain — v4：匯出 + 媒體庫搜尋</project_name>

  <overview>
    PodcastBrain v4 在 v3 的基礎上新增了兩項關鍵的生產力功能：
    匯出選項與含全文搜尋的集數媒體庫。使用者現在可以將任何集數匯出為
    Markdown、PDF、SRT 字幕或純文字格式——也可以一次下載包含所有格式的 ZIP 壓縮檔。
    「集數媒體庫」頁面使用 SQLite FTS5，可在毫秒內搜尋所有集數的逐字稿，
    並支援日期範圍和時長篩選。點擊搜尋結果即可開啟該集數的完整檢視器，
    無需重新處理。這使 PodcastBrain 成為音頻內容的真正個人知識庫，
    而不僅僅是一次性的處理工具。
  </overview>

  <core_features>
    1. 音頻輸入 — URL 文字輸入（YouTube 觀看網址、直接 MP3/M4A/WAV 連結）和
       檔案上傳小工具（.mp3、.m4a、.wav）。yt-dlp 將音頻下載至暫存目錄。
       進度列從 stdout 解析更新。取消按鈕終止子程序。

    2. 本機 Whisper 轉錄 — 模型選擇下拉選單（tiny/base/small/medium/large），
       含每個模型的預估時間和精確度說明。透過 Python API 呼叫 Whisper。
       擷取 full_text 和 segments（start/end/text）。語言自動偵測或由使用者指定。

    3. Claude 章節偵測 — Claude 接收完整逐字稿文字，回傳 JSON 陣列：
       [{title, start_seconds, end_seconds, summary}, ...]。JSON 驗證，最多重試 3 次，
       指數退避。時長短於 300 秒的集數退回至單一章節。

    4. Claude 完整分析 — 三個獨立 Claude API 呼叫：
       (a) generate_summary_quotes_actions(transcript_text, api_key) — JSON {summary, quotes,
           action_items}，其中 quotes 包含 {text, speaker, timestamp_seconds}。
       (b) identify_speakers(transcript_text, api_key) — JSON {speakers: [...]}。
       合併結果以 JSON 形式儲存於 episodes.claude_analysis。

    5. 向集數提問（問答功能） — 關鍵字片段檢索（依比對分數排序前 10 名）。
       Claude 使用「僅根據逐字稿回答」系統提示。來源引用在 st.expander 中展示。
       問答對儲存至 qa_history 資料表。

    6. 匯出選項 — export_markdown(episode_id)、export_pdf(episode_id)、
       export_srt(episode_id)、export_txt(episode_id)。ZIP 使用 Python zipfile 模組。
       Markdown 包含：摘要、章節、引述、逐字稿。PDF 使用 reportlab 含章節標題。
       SRT 從 Whisper 片段生成，格式正確（00:00:00,000）。
       所有匯出透過 st.download_button 提供，使用正確的 MIME 類型。

    7. 含搜尋功能的集數媒體庫 — 透過 st.text_input 對所有集數逐字稿進行 FTS5 全文搜尋。
       以 st.date_input 篩選日期範圍，st.slider 篩選時長（0-180 分鐘）。
       3 欄集數卡片格網：縮圖（YouTube）、標題、時長、處理日期。
       點擊卡片開啟該集數的集數檢視器（從資料庫載入，無需重新處理）。
       snippet() 函式提供搜尋詞出現位置的高亮摘錄。
  </core_features>

  <database_schema>
    -- 主要集數記錄
    CREATE TABLE episodes (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT,
        source_url TEXT NOT NULL,
        source_type TEXT CHECK(source_type IN ('youtube','direct_url','file_upload')),
        audio_path TEXT,
        duration_secs INTEGER,
        whisper_model TEXT,
        language TEXT DEFAULT 'en',
        status TEXT CHECK(status IN ('queued','downloading','transcribing','analyzing','complete','failed')),
        error_message TEXT,
        claude_analysis TEXT,             -- JSON：{summary, quotes, action_items, speakers}
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        processed_at TIMESTAMP
    );

    -- 含片段級時間戳記的完整逐字稿
    CREATE TABLE transcripts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        full_text TEXT NOT NULL,
        segments TEXT,                    -- JSON 陣列：[{start, end, text}, ...]
        word_count INTEGER,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- 來自 Claude 分析的章節標記
    CREATE TABLE chapters (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        title TEXT NOT NULL,
        start_seconds INTEGER NOT NULL,
        end_seconds INTEGER,
        summary TEXT,
        sort_order INTEGER NOT NULL
    );

    -- 每個集數的問答互動歷史記錄
    CREATE TABLE qa_history (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        episode_id INTEGER NOT NULL REFERENCES episodes(id) ON DELETE CASCADE,
        question TEXT NOT NULL,
        answer TEXT NOT NULL,
        source_ts TEXT,                   -- 使用的時間戳記對 JSON 陣列
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    -- FTS5 虛擬資料表（全文逐字稿搜尋）
    CREATE VIRTUAL TABLE transcripts_fts USING fts5(
        episode_id UNINDEXED,
        full_text,
        content='transcripts',
        content_rowid='id'
    );

    -- 觸發器，保持 FTS5 同步
    CREATE TRIGGER transcripts_ai AFTER INSERT ON transcripts BEGIN
        INSERT INTO transcripts_fts(rowid, episode_id, full_text)
        VALUES (new.id, new.episode_id, new.full_text);
    END;
  </database_schema>

  <key_interactions>
    流程 1 — 處理新集數：
      網址 → 處理 → st.status() 顯示四個步驟 → 集數檢視器含 4 個分頁 + 匯出列。

    流程 2 — 提問：
      問答分頁 → 輸入問題 → 帶來源引用的有根據回答（在 st.expander 中）。

    流程 3 — 搜尋集數媒體庫：
      集數媒體庫頁面 → 輸入關鍵字 → FTS5 搜尋所有逐字稿 → 集數卡片
      帶高亮文字摘錄 → 點擊卡片從資料庫開啟完整集數檢視器。

    流程 4 — 匯出：
      任一集數 → 下載按鈕列 → 直接透過瀏覽器下載
      Markdown / PDF / SRT / TXT / ZIP。
  </key_interactions>

    步驟 5 — 問答引擎：（同 v3）

    在 app.py 或 export.py 中：export_markdown(episode_id, db)、export_pdf(episode_id, db)、
    export_srt(episode_id, db)、export_txt(episode_id, db)。

    四分頁佈局（摘要、章節、逐字稿、問答）。
